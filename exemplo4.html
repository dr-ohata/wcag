Problemas de Mensagens de Erro em Formulários

Esta página demonstra problemas comuns relacionados à identificação e correção de erros em formulários.

Importante: Tente enviar os formulários abaixo para ver as diferentes formas de tratar erros.

1. Mensagens de Erro Genéricas ou Inexistente

Problema: Mensagem de Erro Genérica ou Inexistente

Mensagens de erro genéricas como "Corrija os erros abaixo" não ajudam o usuário a entender o que precisa ser corrigido.

Corrija os erros no formulário acima.
<div class="error">Corrija os erros no formulário acima.</div>

Solução: Mensagens de Erro Específicas e Contextuais

As mensagens de erro devem identificar claramente o campo problemático e explicar como corrigir o erro.

Por favor, informe seu nome completo.
Por favor, informe um endereço de e-mail válido.
A senha deve ter pelo menos 8 caracteres.
<input type="text" id="name" aria-describedby="name-error" required>
<span id="name-error" class="error-message">Por favor, informe seu nome completo.</span>

2. Identificação Visual Inadequada dos Campos com Erro

Problema: Apenas Cor para Indicar Erro

Usar apenas cor (como borda vermelha) para identificar campos com erro não é suficiente para usuários daltônicos ou com baixa visão.

Formato inválido
/* Apenas mudança de cor */
input:invalid {
  border-color: #dc3545;
}

Solução: Múltiplos Indicadores de Erro

Além da cor, use ícones, texto e padrões visuais para identificar campos com erro.

Informe um telefone válido com DDD (10 ou 11 dígitos).
/* Múltiplos indicadores */
input[aria-invalid="true"] {
  border: 2px solid #dc3545;
  background-image: url('icon-error.svg');
  background-position: right 10px center;
  background-repeat: no-repeat;
  padding-right: 40px;
}

3. Mensagens de Erro com Linguagem Técnica

Problema: Mensagens Técnicas ou de Sistema

Mensagens de erro com termos técnicos ou códigos de sistema não ajudam o usuário a entender o problema.

ValidationError: Input does not match pattern /[a-zA-Z0-9_]+/
<span class="error">ValidationError: Input does not match pattern /[a-zA-Z0-9_]+/</span>

Solução: Linguagem Clara e Ação Corretiva

Use linguagem simples e direta, explicando o problema e como corrigi-lo.

Use apenas letras, números e underline (_).
<span id="username-error" class="error-message">
  Use apenas letras, números e underline (_).
</span>
